HTTP协议

web常用术语

  • 网页由许多对象组成，对象即文件，包括HTML文件，JPEG图像，音频文件等，每个对象被一个URL寻址

HTTP连接

  • 过程：

    1. 客户初始化一个与HTTP服务器80端口的TCP连接（创建套接字）
    2. HTTP服务器在80端口监听到来自HTTP客户的TCP连接请求，接受，建立连接，通知客户
    3. HTTP客户发送一个HTTP请求信息包含URL到TCP连接套接字，要某个web对象
    4. HTTP服务器接收请求信息，产生一个响应消息包含被请求对象，并发送这个消息到自身TCP连接套接字
    5. 最后关闭TCP连接

  • 分类：

    • 非持久HTTP连接：(HTTP/1.0)

      每个TCP连接只传送一个对象，下载多个对象要建立多个TCP连接

    • 持久HTTP连接：(HTTP/1.1)

      一个TCP连接上可传送多个对象

HTTP报文及方法

cookie

  • 目的：web站点希望能识别用户
  • 四个组件：
    • cookie头部行在HTTP请求信息中
    • cookie头部行在HTTP响应信息中
    • cookie文件保存在用户主机并被用户浏览器管理
    • cookie也保存在web站点的后端数据库
  • 在用户最初发出HTTP请求访问某站点时，该站点创建一个唯一的ID，并在后端数据库创建一个响应于该ID的表项

web缓存（代理服务器）

  • 目的：在不访问服务器的前提下满足客户端的HTTP请求

  • 过程：

    • 用户设定浏览器通过缓存进行Web访问
    • 浏览器向缓存/代理服务器发送所有的HTTP请求
    • 如果所请求对象在缓存中，缓存器返回对象；否则缓存器向起始服务器发出请求，接收对象后转发给客户机

网络交换方式

电路交换

两台主机之间创建一条专用的端到端连接，占用每条链路的一条电路

在一条链路上建立多条连接，每个连接占用链路带宽的1/n

  • 频分多路复用FDM
  • 时分多路复用TDM

分组交换

报文message划分为数据块，封装成分组packet

每个分组使用链路全部带宽

存储转发机制 - 先接收整个分组，再从输出链路转发出去

  • 数据报网络TCP/IP
    • 分组目的地址决定下一跳
    • 会话期间路由可以改变
  • 虚电路网络 - 面向连接
    • 每个分组有1个标签，标签决定下一跳
    • 确定固定的路径，保持于整个会话期间
    • 路由器必须为每个连接维护状态信息

分组交换网络中的延迟、丢失、吞吐量

分组延迟

  • 节点处理时延 - 微秒级，几乎忽略
  • 排队时延 - 等待被发送到输出链路，取决于路由器的拥塞程度
  • 传输时延 - R=链路带宽(bps)，L=分组长度(bits)，t=L/R，类比乘客排队上车
  • 传播时延 - d=物理链路的长度，s=介质的信号传播速度，t=d/s，类比公交车出发到下一个目的地

通常计算端到端时延，一般是传输+传播

分组丢失

输入链路或输出链路缓冲区队列已满，分组被丢弃

吞吐量

接收端接收到数据的比特速率(bps)

min{Rs,Rc}

协议层及其服务模型

  • OSI模型和TCP/IP模型
  • 上到下封装，下到上解封

进程的描述与控制

发表于 2022-02-28 更新于 2022-03-01

前趋图和程序执行

前趋图

  • 有向无循环

程序顺序执行

  • 特征：
    • 顺序性
    • 封闭性
    • 可再现性

程序并发执行

  • 特征：
    • 间断性：具有“执行-暂停-执行”的活动规律
    • 失去封闭性：多个程序共享系统资源
    • 不可再现性

进程的描述

进程的定义和特征

  • 进程是进程实体（PCB、程序段、相关数据段）的运行过程
  • 特征：
    • 动态性：由创建而产生，由调度而执行，由撤销而消亡
    • 并发性：多个进程实体同存于内存，且能同时执行
    • 独立性：进程实体是独立运行、独立分配资源、独立接受调度的基本单位
    • 异步性：进程按各自独立、不可预知的速度前进

进程的基本状态及转换

  • 三种状态：

    • 就绪状态：已分配到除CPU以外的所有必要资源
    • 执行状态：已获得CPU，正在执行
    • 阻塞状态

  • 创建状态：申请空白PCB，填入控制管理信息，分配运行时所必须的资源

    终止状态：PCB清零并返还系统

  • 挂起状态：执行的进程暂停执行

进程管理中的数据结构

  • 进程控制块PCB作用（之一）：实现间断性的运行方式（保护CPU现场）
  • 进程控制块内容：进程标识符、处理机状态、进程调度信息、进程调度信息、进程控制信息
  • 进程控制块的组织方式：多级链接

进程控制

操作系统内核

进程控制一般是由OS内核中的一组原语实现的

进程的创建

  • 进程图
  • 引起创建进程的事件：用户登录、作业调度、提供服务、应用请求
  • 进程的创建：
    1. 申请空白的PCB
    2. 为新进程分配资源
    3. 初始化进程控制块各种信息
    4. 将新进程插入就绪队列

进程的终止

  • 引起进程终止的事件：正常结束、异常结束、外界干预
  • 进程的终止过程：
    1. 根据标识符检索出PCB，读取进程状态
    2. 若该进程处于执行状态，立即终止执行，重新进行调度
    3. 若有子孙进程，将其所有子孙进程终止
    4. 将其全部资源归还给其父进程或系统
    5. 将PCB从所在队列中移除

进程的阻塞与唤醒

  • 进程阻塞过程：
    1. 进程调用阻塞原语block将自己阻塞
    2. 将现行状态由执行改为阻塞，并将PCB插入阻塞队列
    3. 转调度程序进行重新调度
  • 进程唤醒过程：
    1. 有关进程调用唤醒原语wakeup()
    2. 将进程从阻塞队列中移出，将现行状态改为就绪，然后重新插入到就绪队列

进程的挂起与激活

进程同步

进程同步的基本概念

  • 进程同步机制：使并发执行的诸进程按一定规则共享系统资源

  • 某些资源必须互斥使用，这类资源称为临界资源，访问临界资源的代码称为临界区

    任何时刻，只允许一个进程进入临界区，以实现进程对临界资源的互斥访问

操作系统引论

发表于 2022-02-23

1.1 基本概念

关于现代OS的四种观点

  • 作为用户与计算机硬件系统之间的接口
  • 是建立在计算机硬件平台上的虚拟机器（计算机资源的抽象）
  • 作为计算机系统资源的管理者
  • 是计算机系统工作流程的组织者

1.2 OS的系统需求

OS的功能性需求

  • 计算机用户需要的用户命令（用户接口）
  • 应用软件需要的系统调用（程序接口，即API）

OS的非功能性需求

  • 性能：吞吐量、响应时间、用户数……
  • 公平性：如对某些应用是否有偏向（不同系统要求不同）
  • 可靠性：结果的是否准确
  • 安全性
  • 可伸缩性：合理的预算购买适量的功能
  • 可扩展性：对不同设备的支持
  • 可移植性：对平台的要求
  • ……

OS对硬件平台的依赖

  • 计时器、I/O中断、DMA、特权指令、存储保护机制……（均由硬件实现）

重要的基本概念

  • 作业 - 用户一次上机过程中要求计算机系统所做的工作集合，每项相对独立的工作称为作业步

  • 线程 - 程序一次相对独立的运行过程（动态概念），线程是系统调度的最小单位

    进程 - 系统分配资源的基本对象，是系统中拥有资源的最小实体

    ps：相当于搬运工与仓库

  • 虚拟存储 - 进程的逻辑地址空间，

  • 文件 - 命名了的字节流，该概念非常宽泛

1.3 OS的演变、类型及特征

单道批处理系统

  • 对作业的处理是成批进行，但内存中只保持一道作业
  • 主要缺点 - CPU必须等待I/O，速度不匹配，CPU利用率降低

多道批处理系统

  • 允许多个作业调入内存
  • 一些问题 - 保证同步（如存取数据需要启动信号）、正确的互斥（对共享资源的使用）、运行的确定性（运行环境的影响）、死锁

分时系统

  • 多个用户分享处理器的时间，多用户通过多终端访问系统，主要满足交互需求

1.4 现代OS的基本类型

单机OS

  • 批处理系统
  • 分时系统
  • 实时系统

现代OS的两个基本特征

  • 任务共行 - 宏观上是系统中多个任务同时运行，微观上指单处理机系统中的任务并发
  • 资源共享 - 宏观上是多个任务同时使用系统的软硬件资源，微观上是多个任务交替互斥使用系统中某个资源

1.5 OS的体系结构

常见的OS总体结构风格

  • 大多数包含两类子系统：
    • 用户接口子系统：提供用户命令
    • 基础平台子系统：提供系统调用
    • 上到下单向关系

基础平台子系统结构

  • 分层 - 每一层都对其上各层隐藏其下层的存在
  • 分级 - 所有各级实现只依赖以下各级所提供的概念及属性
  • 分块 - 所有各块的实现都可任意引用其余各块

基础平台子系统结构（二）

  • 多模式结构风格 - 不同的模式运行在不同的CPU特权模式下
  • 单模式结构风格 - 仅包含一个模式模块，由应用软件和基础平台子系统共同构成

基础平台子系统结构（三）

  • 双模式基础平台子系统：两个模式模块，即核外子系统和核心子系统
  • 微核结构：尽量减少核心子系统的大小

1.1 数据库及其系统的概念

1.1.1 数据库的定义

  • 定义：依照特定数据模型组织、存储和管理数据的文件集合
  • 数据库文件特点：
    • 数据一般不重复存放
    • 可支持多个应用程序并发访问
    • 数据结构独立于使用它的应用程序
    • 增删查改操作均由数据库系统软件进行管理和控制

1.1.2 数据模型

  • 定义：一种描述事物对象数据特征及其结构的形式化表示

  • 组成：

    • 数据结构：描述事物对象的静态特征，包括事物对象的属性数据、数据类型、数据组织方式等
    • 数据操作：描述事物对象的动态特征，如增删查改等
    • 数据约束：描述事物对象的数据之间语义的联系，以及数据取值范围等规则，确保数据的完整性、一致性和有效性

  • 数据库前后使用的数据模型（根本区别在于数据之间的联系方式不同）

    • 层次数据模型：（树结构）

    • 网状数据模型：（图结构）

    • 关系数据模型：（关系二维表——表之间通过具有相同列属性的数据值进行约束关联）

1.1.3 数据库系统的组成

  • 用户：分为最终用户和DBA用户

  • 数据库应用程序：使用DBMS提供的标准接口

  • 数据库管理系统：属于系统软件

  • 数据库：存放系统各类数据

    以关系数据库为例：用户数据存放在用户表，元数据、索引数据、运行数据等存放在系统表
